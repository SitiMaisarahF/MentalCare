This portfolio showcases a comprehensive learning journey in Android mobile application development undertaken throughout the TTTM2213 course. Over the semester, development progressed from implementing foundational user interfaces to integrating complex architectures, local persistent storage, and hardware sensors. The practical skills culminated in the creation of 'MentalCare,' a dedicated application designed to support personal well-being and streamline mental health resource tracking.

Labs

Lab 1 (UI)

Learned the fundamentals of constructing modern, declarative user interfaces using Jetpack Compose. This lab emphasized layout components like Rows, Columns, and Boxes to align UI elements correctly while ensuring adaptivity across varying screen dimensions.

Lab 2 (Interaction)

Focused on managing interactive states, processing user input events, and dynamically modifying the UI in response to user actions. Gained a solid understanding of how state hoisting and mutable states drive reactivity in modern Android development.

Lab 3 (Material Design)

Integrated official Material 3 Design principles to standardize color themes, typography, and shape styling across the application components. This structure improved the overall visual aesthetics and provided a highly consistent, professional user experience.

Lab 4 (Navigation)

Implemented multi-screen navigation using the Jetpack Compose Navigation component, establishing a clean, type-safe navigation graph. Mastered the process of passing data arguments securely between destination routes and handling the back stack effectively.

Lab 5 (Room)

Explored local persistence by integrating the Room database library to execute robust CRUD operations on structured data. Successfully managed Data Access Objects (DAOs), database entities, and asynchronous data streams using Kotlin Coroutines and Flow.

Projects

MentalCare

Problem Statement: Individuals seeking to improve their mental health often lack a consolidated, private application to log daily moods and organize immediate coping resources, resulting in inconsistent self-monitoring habits.

Developed core architectural workflows by connecting basic multi-screen navigation to operational components. Learned the importance of structured application state management and how to handle user interface inputs without degrading app performance.

MentalCare (Advanced)

Problem Statement: Static wellness tracking apps lack real-time context and secure cloud backup, making it difficult for users to access synchronized mental health records or engage interactively with their immediate environmental surroundings.

Successfully integrated hardware sensors to track contextual user movement alongside RESTful web APIs to fetch external health tips dynamically. Overcoming configuration challenges with cloud synchronization significantly enhanced my architectural debugging skills and understanding of background threads.

Reflection

This entire course provided an immersive hands-on evolution from a foundational learner to an adaptable mobile application developer capable of building complex Android architectures. Overcoming technical hurdles related to local database reactivity and cloud-based data integration taught me how to read comprehensive documentation and debug system logs systematically. Developing the MentalCare application solidified my engineering mindset, teaching me to prioritize clean architecture and user-centric design when mapping technology to real-world social impact goals.
